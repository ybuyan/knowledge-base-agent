# 公司制度知识库问答与流程指引助手

## 核心需求
构建一个公司制度知识库问答与流程指引助手，支持员工基于员工手册、请假制度、考勤规范、报销流程、入离职流程和常见FAQ进行自然语言问答。系统需要围绕基础知识库构建与检索增强问答展开，能够对制度文档进行切分、向量化、索引和检索，并结合大语言模型生成有依据的回答。系统应重点体现向量检索、RAG问答、多轮对话和引用溯源等基础能力，而不是构建复杂的知识平台或完整业务系统。

本题更强调知识库问答原型的可用性，不要求实现复杂的混合检索、重排序、在线增量更新或企业级知识治理能力，这些内容可以作为扩展项。

## 使用场景示例

**制度问答：**
- 用户提出“年假怎么计算？”“试用期请假有什么规定？”“迟到会怎么处理？”等问题。
- 系统从员工手册或公司制度文档中检索相关条款，并基于检索结果生成准确、简洁的回答。
- 系统在回答中标注制度依据和来源位置，便于用户核对。

**多轮追问：**
- 用户在上一轮提问后继续追问“病假和年假有什么区别？”“那如果是试用期员工呢？”等问题。
- 系统能够理解上下文，保持对当前制度主题和问题对象的连续理解。
- 系统能够在基础多轮对话中持续给出有依据的回答，而不是脱离上下文重新泛化作答。

**流程与材料指引：**
- 用户提出“请假需要提交什么材料？”“入职需要准备哪些文件？”“报销流程怎么走？”等问题。
- 系统根据制度文档和流程说明返回对应的材料清单、表单要求和办理步骤。
- 系统能够区分不同事项类型对应的差异化要求，如病假、婚假、报销、入职等。

**引用与无答案处理：**
- 系统在回答中展示命中的制度条款、文档名称和相关段落。
- 当知识库中没有足够依据时，系统能够明确说明“不确定”或“未找到依据”，避免编造答案。
- 系统可以提示用户补充问题背景，或引导其联系HR进一步确认。

**FAQ与知识沉淀：**
- 系统整理常见员工问题及标准答案，形成可复用的问答知识库。
- 系统识别高频问题，为HR优化制度说明和员工服务流程提供参考。
- 系统可以基于新增FAQ或补充文档优化知识库内容。

**功能需求**：
1. 系统能够接收和处理员工手册、制度文档、流程文档和FAQ数据。
2. 系统能够理解用户的自然语言问题，并识别问题所属场景，如请假、考勤、报销、入职、离职或转正。
3. 系统能够对制度文档进行切分、向量化并建立知识库索引。
4. 系统能够基于向量检索，从知识库中召回相关内容并生成有依据的回答。
5. 系统能够为用户提供流程步骤、责任角色、所需材料和注意事项。
6. 系统能够在回答中给出来源说明、条款引用或文档位置。
7. 系统能够支持基础多轮对话，处理用户基于前文的追问。
8. 系统能够在知识不足时进行基础兜底处理，如明确提示未找到依据、请求补充信息或建议联系HR确认。
9. 系统应该提供一个简洁的问答界面，便于员工或HR人员使用。

**数据源**：
- 员工手册与制度文档：包括员工手册、请假制度、考勤规范、差旅报销规定、行为规范等，可使用脱敏文档或参赛者自行构造的模拟文档。
- 流程文档：包括请假申请、报销申请、入职办理、离职审批、转正流程等操作指引，可使用样例文档或模拟流程说明。
- FAQ数据：可使用已有员工服务FAQ，或由参赛者基于制度文档自行构造。
- 文档元数据：包括文档名称、版本、生效日期、适用范围、所属部门等，可用于过滤和引用。
- 组织与角色信息：可使用部门、岗位和审批角色的样例数据辅助回答。

**提交要求**：
- 完整的源代码。
- 部署和运行的说明文档。
- 一份演示报告，展示系统在制度问答、多轮追问、引用展示和无答案处理场景下的表现。

**附录：技术参考**
文档处理
- PyPDF2 / pdfplumber：用于处理PDF制度文档。
- python-docx：用于处理Word格式的制度和流程文件。
- pytesseract / EasyOCR：用于识别扫描件中的文字内容。
检索与问答
- LangChain / LlamaIndex：用于构建检索增强问答流程。
- FAISS / Chroma：用于向量检索和知识库索引。
- BGE / text-embedding-3 等Embedding模型：用于文档向量化。
- LLM API：用于问题理解、答案生成和多轮问答。
数据处理
- Pandas：用于处理FAQ、文档元数据和辅助结构化数据。
- regex：用于提取条款编号、流程节点和关键字段。
可视化与应用
- Streamlit / Dash：用于构建制度问答和流程指引界面。
