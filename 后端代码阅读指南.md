## 三、核心模块详解

### 1. 应用启动流程

**文件：** `app/main.py`

启动顺序：
1. 注册 CORS 中间件
2. 注册 X-Trace-ID 中间件
3. 注册全局异常处理器
4. 执行 startup 事件：
   - `reset_collections()`：检测 ChromaDB 维度
   - `init_chroma()`：初始化向量存储
   - `connect_to_mongo()`：连接 MongoDB
   - 注册 7 个 Tool 实现
   - 注册 5 个 Agent
   - 创建流程指引索引
   - 启动后台文件扫描器
   - 注册 MCP Server
   - 启动 MCP 审计日志
   - 启动提示词同步任务

### 2. 聊天核心流程

**文件：** `app/services/qa_agent.py`

**核心方法：** `process()`

处理流程：
1. **禁止主题检查**：字符串匹配 + LLM 语义检查
2. **查询优化**：`KBQueryOptimizer.optimize()`
3. **工具决策**：`LLMClient.chat_with_tools()`
4. **ReAct 循环**（最多 5 轮）：
   - `ToolExecutor.execute_batch()` 并行执行工具
   - `LLMClient.chat_with_tools()` 决定下一步
5. **生成答案**：`LLMClient.stream_chat()`
6. **后处理**：
   - `SuggestionGenerator.generate()`：生成 3 个快捷提问
   - `LinkMatcher.match_links()`：匹配外部链接
   - `ResponseBuilder.done_chunk()`：构建最终响应

### 3. 向量存储管理

**文件：** `app/core/chroma.py`

**核心功能：**
- 自动检测 embedding 维度不匹配
- 智能重建集合（避免数据丢失）
- 支持 6 种常见 embedding 模型维度

**支持的维度：**
- text-embedding-v1: 1536
- text-embedding-v2: 1536
- text-embedding-v3: 1024
- text-embedding-ada-002: 1536
- text-embedding-3-small: 1536
- text-embedding-3-large: 3072

### 4. 文档处理

**文件：** `app/api/routes/documents.py`

**功能：**
- 多格式文档上传（PDF、Word、Excel、PowerPoint、图像）
- 后台文件扫描器（30 秒轮询）
- 文档索引与向量存储

**处理流水线：**
```
DocumentParser → TextSplitter → EmbeddingProcessor → VectorStore
  chunk_size=500, chunk_overlap=50, split_by=paragraph
```

### 5. Prompt 系统

**文件：** `app/prompts/config.json`

**核心提示词：**
- `qa_rag`：RAG 问答
- `strict_qa`：严格知识库问答
- `fallback`：无结果兜底
- `tool_router`：工具路由决策
- `flow_extract`：流程提取
- `query_optimize`：查询优化

## 九、常见问题

### 1. 维度不匹配错误
**错误：** `Embedding dimension 1536 does not match collection dimensionality 1024`
**解决：** 系统会自动检测并重建集合，无需手动操作。

### 2. MongoDB 连接失败
**错误：** `MongoDB 连接失败，将使用纯短期记忆模式`
**解决：** 检查 MongoDB 服务是否运行，配置是否正确。

### 3. 文档处理失败
**错误：** `文档处理失败: xxx.pdf`
**解决：** 检查文件格式是否支持，文件是否损坏。

### 4. 提示词渲染错误
**错误：** `KeyError: '\n    "name"'`
**解决：** 检查提示词模板中的花括号转义。

## 十、扩展指南

### 新增 Tool
1. 在 `app/tools/implementations/` 中创建新的 Tool 类
2. 在 `app/main.py` 的 startup 事件中注册
3. 确保实现 `execute()` 方法

### 新增 Agent
1. 在 `app/agents/implementations/` 中创建新的 Agent 类
2. 在 `app/main.py` 的 startup 事件中注册
3. 实现 `run()` 方法

### 新增 Prompt
1. 在 `app/prompts/config.json` 中添加新的提示词
2. 重启应用或等待定时同步
