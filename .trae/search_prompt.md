# 查询优化功能改进计划

## 目标

提升查询优化功能的性能、准确性和用户体验，降低API调用成本。

## 当前状态

- 使用通义千问(qwen-plus)进行查询重写
- 基于Prompt Engineering + Few-shot Learning
- 每次查询都调用LLM API
- 无缓存机制

## 优化计划

### 阶段一：性能优化（优先级：高）

#### 1.1 添加查询缓存
- **目标**: 减少重复API调用，降低成本和延迟
- **实现**:
  - 使用Redis缓存优化结果
  - 缓存键：原始查询的hash值
  - 缓存过期时间：24小时
- **预期效果**: 相同查询响应时间从~1s降至~50ms

#### 1.2 异步处理优化
- **目标**: 提升并发处理能力
- **实现**:
  - 添加请求队列
  - 批量处理优化请求
- **预期效果**: 支持更高并发

### 阶段二：算法优化（优先级：中）

#### 2.1 混合优化策略
- **目标**: 结合传统NLP和LLM，提升效果
- **实现**:
  - 使用jieba提取关键词
  - 使用TF-IDF计算词权重
  - LLM进行语义扩展
- **预期效果**: 关键词提取更准确

#### 2.2 查询分类
- **目标**: 根据查询类型采用不同优化策略
- **实现**:
  - 事实型查询：强调关键词
  - 流程型查询：强调步骤词
  - 概念型查询：强调定义和解释
- **预期效果**: 优化结果更精准

#### 2.3 领域适配
- **目标**: 针对企业文档领域优化
- **实现**:
  - 构建企业术语词典
  - 添加领域特定Prompt
  - 微调小模型（可选）
- **预期效果**: 行业术语理解更准确

### 阶段三：用户体验优化（优先级：中）

#### 3.1 优化预览
- **目标**: 让用户了解优化效果
- **实现**:
  - 显示原始查询vs优化查询对比
  - 高亮显示新增关键词
- **预期效果**: 用户信任度提升

#### 3.2 多候选结果
- **目标**: 提供多个优化选项
- **实现**:
  - 返回3个优化版本
  - 用户选择最适合的
- **预期效果**: 用户满意度提升

#### 3.3 反馈机制
- **目标**: 收集用户反馈，持续改进
- **实现**:
  - 添加"有用/无用"反馈按钮
  - 记录优化前后检索结果对比
- **预期效果**: 数据驱动优化

### 阶段四：高级功能（优先级：低）

#### 4.1 多轮优化
- **目标**: 根据检索结果迭代优化
- **实现**:
  - 检测检索结果质量
  - 自动调整优化策略
- **预期效果**: 检索准确率提升

#### 4.2 智能推荐
- **目标**: 推荐相关问题
- **实现**:
  - 基于优化结果生成相关问题
  - 显示在输入框下方
- **预期效果**: 引导用户提问

## 技术栈

- **缓存**: Redis
- **NLP**: jieba, scikit-learn
- **LLM**: 通义千问 (qwen-plus/turbo)
- **前端**: Vue 3 + Element Plus

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| LLM API不稳定 | 高 | 添加降级策略，使用关键词提取 |
| 缓存命中率低 | 中 | 设置合理的过期时间，定期清理 |
| 优化结果不准确 | 中 | 添加用户反馈机制 |

## 成功指标

- 缓存命中率 > 30%
- 优化响应时间 < 500ms（缓存命中时 < 100ms）
- 用户满意度 > 80%
- 检索准确率提升 > 20%
