# 多轮对话请假指引升级

## 升级内容

### 1. Triggers 的新作用

**之前**: Triggers 用于 Orchestrator 的意图识别（已被 LLM 替代）

**现在**: Triggers 用于 GuideAgent 匹配具体的 guide skill

**作用场景**:
- 当有多个 guide skill 时，用于区分使用哪个
- 例如：
  - `leave_guide` - triggers: ["请假", "休假", "年假"]
  - `reimbursement_guide` - triggers: ["报销", "费用"]
  - `onboarding_guide` - triggers: ["入职", "新员工"]

**工作流程**:
```
用户: "请假申请怎么写"
  ↓
Orchestrator (LLM): 识别为 guide 意图
  ↓
GuideAgent: 遍历所有 guide 类型的 skills
  ↓
匹配 triggers: "请假" in query → 找到 leave_guide
  ↓
执行 leave_guide skill
```

### 2. 多轮对话优化

#### 新的对话流程（6个阶段）

**阶段 1: 欢迎与假期类型确认**
- 提供清晰的选项列表
- 友好的欢迎语

**阶段 2: 请假天数确认**
- 根据假期类型提供参考信息
- 告知额度和建议

**阶段 3: 时间确认**
- 询问具体开始时间
- 支持多种时间表达

**阶段 4: 特殊情况确认**
- 病假：是否有病假条
- 婚假：结婚登记日期
- 产假：预产期
- 高温假：工作性质

**阶段 5: 信息汇总确认**
- 汇总所有收集的信息
- 让用户确认或修改

**阶段 6: 生成个性化指引**
- 根据收集的信息定制内容
- 包含完整的申请指引

#### 个性化内容

生成的指引包含：
1. **假期情况总结** - 类型、天数、时间、额度
2. **准备材料** - 根据假期类型定制
3. **申请步骤** - 详细的操作指南
4. **审批流程** - 根据天数调整流程
5. **申请模板** - 个性化的申请模板
6. **注意事项** - 针对性的提醒
7. **联系方式** - HR 和 IT 支持

### 3. 智能对话策略

#### 渐进式收集
- 一次只问一个问题
- 避免信息过载
- 循序渐进

#### 智能理解
自动识别和提取信息：
- "我想请3天假" → 提取天数
- "下周一到周三" → 计算天数
- "6月15号开始，请一周" → 计算结束日期

#### 主动提示
- 提供选项帮助选择
- 给出参考信息
- 提醒注意事项

#### 确认机制
- 收集完信息后汇总
- 允许用户修改
- 确认无误后生成

#### 个性化定制
- 根据假期类型定制内容
- 根据天数调整审批流程
- 根据时间提醒注意事项

## 扩展性

### 添加新的 guide skill

1. 创建新的 SKILL.md
2. 设置 skill_type: guide
3. 定义 triggers（区分不同 skill）
4. 创建对应的 prompt 模板
5. GuideAgent 自动识别和执行

### 示例：报销指引

```yaml
---
name: reimbursement_guide
skill_type: guide
triggers:
  - 报销
  - 费用
  - 发票
---
```
