# 需求文档：MCP 资源公开访问（mcp-resource-public-access）

## 概述

为现有 MCP 标准实现添加认证与授权机制，使静态资源知识库可以安全地对外开放访问。当前 MCP 端点没有认证机制，不适合对外开放。本需求旨在引入完整的安全机制，同时保持与 MCP 协议规范的兼容性。

## 业务需求

### BR-1: API Key 认证
**优先级**: 高  
**描述**: 系统必须支持基于 API Key 的认证机制，允许外部用户通过 API Key 访问 MCP 资源。

**验收标准**:
- 用户可以生成、管理和撤销 API Key
- API Key 格式为 `mcp_<64位十六进制字符>`
- API Key 支持过期时间配置
- 无效或过期的 API Key 被拒绝访问

### BR-2: 资源分级访问控制
**优先级**: 高  
**描述**: 系统必须支持三级资源分类（公开、内部、机密），并根据用户角色控制访问权限。

**验收标准**:
- 公开资源：所有用户可访问
- 内部资源：仅员工和管理员可访问
- 机密资源：仅具有特殊权限的用户可访问
- 资源 URI 遵循命名约定：`<scheme>://<level>/<category>/<resource_name>`

### BR-3: 审计日志
**优先级**: 高  
**描述**: 系统必须记录所有 MCP 访问行为，包括成功和失败的请求。

**验收标准**:
- 记录用户身份、访问时间、资源 URI、操作结果
- 敏感信息（完整 API Key）脱敏处理
- 支持查询用户访问历史和资源访问统计
- 日志保留 90 天

### BR-4: 速率限制
**优先级**: 高  
**描述**: 系统必须防止 API 滥用和 DDoS 攻击，对每个用户实施速率限制。

**验收标准**:
- 基于用户的速率限制（每分钟请求数可配置）
- 基于 IP 的全局速率限制
- 超出限制返回 429 Too Many Requests
- 提供剩余配额查询

### BR-5: 公开资源目录
**优先级**: 中  
**描述**: 系统必须提供公开资源目录，让外部用户可以浏览可用资源。

**验收标准**:
- 无需认证即可访问公开资源列表
- 返回资源 URI、名称、描述等信息
- 支持分类浏览

### BR-6: 管理员功能
**优先级**: 中  
**描述**: 系统必须提供管理员功能，用于管理 API Key 和查询审计日志。

**验收标准**:
- 管理员可以创建、撤销 API Key
- 管理员可以查询审计日志
- 管理员可以查看资源访问统计
- 需要 admin 角色权限

## 功能需求

### FR-1: 认证中间件
**描述**: 实现认证中间件，拦截所有 MCP 请求并验证身份。

**输入**: HTTP Request（包含 X-API-Key 或 Authorization 头）  
**输出**: AuthContext（用户身份和权限信息）

**功能点**:
- 支持 API Key 认证（Header: X-API-Key）
- 支持 OAuth2 Token 认证（Header: Authorization: Bearer <token>）
- 无认证凭证时返回 guest 用户上下文
- 认证失败记录到审计日志

### FR-2: API Key 管理
**描述**: 实现 API Key 的生成、验证、撤销功能。

**功能点**:
- 生成安全的随机 API Key（256 位熵）
- 验证 API Key 有效性（存在、未过期、未撤销）
- 撤销 API Key（立即生效）
- 列出用户的所有 API Keys
- 自动清理过期的 API Keys

### FR-3: 资源访问控制
**描述**: 实现基于资源级别和用户角色的访问控制。

**功能点**:
- 从资源 URI 解析访问级别
- 检查用户是否有权访问指定资源
- 过滤资源列表（只返回用户有权访问的资源）
- 访问决策记录到审计日志

### FR-4: 审计日志记录
**描述**: 实现审计日志记录功能，记录所有 MCP 访问行为。

**功能点**:
- 记录访问事件到 MongoDB
- 敏感信息脱敏（API Key 只记录前 8 位）
- 查询用户访问历史
- 获取资源访问统计
- 异步批量写入（不阻塞主流程）

### FR-5: 速率限制
**描述**: 实现速率限制功能，防止 API 滥用。

**功能点**:
- 基于滑动窗口算法的速率限制
- 检查用户是否超出速率限制
- 获取用户剩余配额
- 重置用户配额（管理员操作）
- 使用 Redis 或内存存储请求计数

### FR-6: 增强的 MCP Dispatcher
**描述**: 在现有 Dispatcher 基础上集成认证、授权、审计、速率限制。

**功能点**:
- 速率限制检查
- 访问权限检查（针对 resources/* 方法）
- 审计日志记录
- 性能监控
- 资源列表过滤（根据用户权限）

### FR-7: 增强的 MCP Router
**描述**: 在路由层集成认证中间件。

**功能点**:
- 依赖注入获取认证上下文
- 公开资源目录端点（GET /mcp/public/resources）
- 管理员 API Key 管理端点（POST/DELETE /mcp/admin/api-keys）
- 管理员审计日志查询端点（GET /mcp/admin/audit-logs）

## 非功能需求

### NFR-1: 性能
- 认证延迟 < 10ms (P95)
- 资源访问延迟 < 100ms (P95)
- 速率限制检查 < 5ms
- 审计日志异步写入不阻塞主流程

### NFR-2: 安全
- API Key 使用加密安全的随机数生成
- 密钥长度 64 位十六进制（256 位熵）
- 敏感信息脱敏处理
- 防止 SQL/NoSQL 注入
- 防止重放攻击

### NFR-3: 可扩展性
- 支持 Redis 缓存（提升认证性能）
- 支持水平扩展（无状态设计）
- 支持高并发访问

### NFR-4: 可维护性
- 代码结构清晰，职责分离
- 完整的单元测试和集成测试
- 详细的 API 文档
- 完善的错误处理和日志记录

### NFR-5: 兼容性
- 保持与 MCP 协议规范的完整兼容性
- 不破坏现有 MCP Server 功能
- 向后兼容现有资源 URI

## 数据需求

### DR-1: MongoDB 集合

#### api_keys 集合
- key: 唯一索引
- user_id: 普通索引
- is_active: 普通索引
- expires_at: TTL 索引

#### audit_logs 集合
- timestamp: 降序索引
- user_id: 普通索引
- resource_uri: 普通索引
- success: 普通索引
- 复合索引: (user_id, timestamp)
- TTL 索引: 90 天自动清理

#### rate_limit_records 集合（可选）
- user_id: 唯一索引
- expires_at: TTL 索引

### DR-2: 资源 URI 命名约定
- 格式: `<scheme>://<level>/<category>/<resource_name>`
- scheme: knowledge | document
- level: public | internal | confidential
- category: policy | guide | faq | manual

## 风险和缓解

### R-1: 性能风险
**风险**: 认证和授权检查可能影响性能  
**缓解**: 使用 Redis 缓存，优化数据库查询，异步处理审计日志

### R-2: 安全风险
**风险**: API Key 泄露导致未授权访问  
**缓解**: 支持即时撤销，设置过期时间，监控异常访问

### R-3: 兼容性风险
**风险**: 新功能可能破坏现有功能  
**缓解**: 充分的集成测试，渐进式迁移，提供回滚方案

### R-4: 复杂性风险
**风险**: 系统复杂度增加，维护成本上升  
**缓解**: 清晰的代码结构，完善的文档，充分的测试覆盖
