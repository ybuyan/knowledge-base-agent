# 技术设计文档 - 流程指引功能

## 概述

流程指引功能是对现有知识库问答助手的重要升级，从"知识查询式"向"流程引导式"演进。本设计基于现有的Agent架构（OrchestratorAgent、GuideAgent、QAAgent等）和Skill引擎，通过扩展和增强实现流程指引能力。

### 设计目标

1. 智能识别用户的流程类意图，自动触发流程指引
2. 通过多轮对话收集用户信息，按步骤引导用户完成操作
3. 提供可视化配置工具，让HR用户能够自主维护流程模板
4. 与业务系统无缝集成，实现流程闭环
5. 保持与现有系统的兼容性，最小化对现有代码的影响

### 核心特性

- 基于LLM的意图识别，准确率>=90%
- 流程状态机管理，支持复杂分支逻辑
- 实时信息校验，提升用户体验
- 流程上下文持久化，支持中断恢复
- 完整的监控和数据分析能力



## 系统架构

### 整体架构

系统采用前后端分离架构，后端基于FastAPI + MongoDB + ChromaDB，前端基于Vue 3 + TypeScript。流程指引功能作为新模块集成到现有系统中。

### 架构说明

1. **前端层**: 负责用户交互，包括聊天界面、流程卡片展示、配置门户
2. **API网关层**: 提供RESTful API和WebSocket接口，处理请求路由和认证
3. **Agent编排层**: 基于现有Agent架构，OrchestratorAgent负责意图识别和路由，GuideAgent负责流程指引
4. **核心服务层**: 包含流程引擎、技能引擎、校验器等核心组件
5. **数据存储层**: MongoDB存储业务数据，ChromaDB用于向量检索，Redis用于缓存和会话管理

### 技术栈

**后端**:
- FastAPI: Web框架
- Python 3.9+: 编程语言
- MongoDB: 主数据库（会话、消息、流程配置、流程上下文）
- ChromaDB: 向量数据库（知识库检索）
- Redis: 缓存和会话管理
- Motor: 异步MongoDB驱动
- Pydantic: 数据验证

**前端**:
- Vue 3: 前端框架
- TypeScript: 类型安全
- Vite: 构建工具
- Element Plus: UI组件库
- Axios: HTTP客户端

**AI/ML**:
- OpenAI API: LLM服务（意图识别、内容生成）
- Sentence Transformers: 文本嵌入
