# 请假指引功能说明

## 功能概述

用户通过对话询问"我想请假"，AI 助手会：
1. 通过自然对话收集请假信息（类型、天数、时间）
2. 生成一份详细的操作指引文档
3. 告诉用户具体怎么去申请请假

## 与流程执行的区别

| 特性 | 流程执行（已删除） | 指引生成（当前） |
|------|-------------------|------------------|
| 交互方式 | 流程卡片表单 | 自然对话 |
| 最终输出 | 直接提交申请 | 生成操作指引 |
| 用户操作 | AI 完成所有操作 | 用户根据指引自己操作 |
| 适用场景 | 系统集成 | 咨询、指导 |

## 实现方式

使用 `skill_type: qa`，利用 LLM 的对话能力：
- 无需开发新的 Processor
- 通过 SKILL.md 中的指引让 AI 知道如何回答
- AI 自动处理多轮对话和信息收集

## 对话示例

```
用户：我想请假
AI：好的，我来帮您了解请假流程。请问您要申请什么类型的假期呢？
    我们有年假、病假、事假、婚假、产假、陪产假和高温假。

用户：年假，3天，从6月15日开始
AI：明白了！让我为您生成请假操作指引。

    📊 假期余额
    您当前剩余年假 5 天，可申请 3 天年假。

    📝 申请流程
    1. 发起请假申请
    2. 直属上级审批
    3. HR 备案

    🔗 申请入口
    https://oa.company.com/leave

    📄 参考模板
    【年假申请】
    申请人：张三
    时间：2024 年 06 月 15 日 - 2024 年 06 月 17 日，共 3 天
    事由：个人事务

    ⚠️ 注意事项
    - 请提前 3 天申请
    - 审批时间：1-3 个工作日
    - 紧急情况请先电话联系直属上级

    如有其他问题，随时问我！
```

## 文件位置

- Skill 定义：`backend/app/skills/definitions/leave_guide/SKILL.md`
- 触发词：我想请假、怎么请假、请假流程、如何申请假期

## 自定义指引内容

如需修改指引内容（如申请入口、流程步骤、注意事项），编辑 `SKILL.md` 文件中的模板部分即可。

## 已删除的内容

- ❌ `leave_apply` skill（流程执行）
- ❌ `nodes.json`（流程节点配置）
- ❌ 流程卡片 UI
- ❌ 相关测试脚本和文档

## 优势

1. **简单**：无需复杂的流程配置
2. **灵活**：AI 自动适应不同的对话方式
3. **易维护**：只需修改 SKILL.md 中的文本
4. **自然**：对话式交互更友好
